# 🧠 Memory System & Instruction Decoding (MIPS)

This document describes the three key memory components used in the Single-Cycle MIPS processor and demonstrates how instructions are decoded and interpreted.

---

## 📦 Memory Units in MIPS

### 📄 Instruction Memory
**Definition:**  
Instruction Memory stores the program in the form of machine code (32-bit instructions).

**Role:**  
- Supplies instructions based on the Program Counter (PC)  
- Acts as a read-only memory during execution  
- Drives the control flow of the processor  

---

### 📄 Data Memory
**Definition:**  
Data Memory holds data values that are accessed and modified during execution.

**Role:**  
- Used by `lw` (load) and `sw` (store) instructions  
- Supports both read and write operations  
- Address is generated by the ALU  

---

### 📄 Register File
**Definition:**  
The Register File is a small, high-speed storage inside the CPU consisting of 32 registers.

**Role:**  
- Stores operands required for computation  
- Supplies inputs to the ALU  
- Stores final results after execution (write-back stage)  

---

## 🔍 Instruction Decoding

Each MIPS instruction is 32 bits long and is divided into fields:

| Field     | Bits      |
|----------|----------|
| opcode   | [31:26]  |
| rs       | [25:21]  |
| rt       | [20:16]  |
| rd       | [15:11]  |
| funct    | [5:0]    |
| immediate| [15:0]   |

The **opcode** determines the instruction type, while the **funct field** specifies the exact operation for R-type instructions.

---

## 🔷 R-Type Instructions (opcode = `000000`)

### `02328020` → `add r16, r17, r18`
- rs = 17, rt = 18, rd = 16  
- funct = `100000` → ADD  

---

### `016d4022` → `sub r8, r11, r13`
- funct = `100010` → SUB  

---

### `012a4024` → `and r8, r9, r10`
- funct = `100100` → AND  

---

### `012a4025` → `or r8, r9, r10`
- funct = `100101` → OR  

---

### `012a402a` → `slt r8, r9, r10`
- funct = `101010` → SLT  

---

## 🔷 I-Type Instructions

### `8d280004` → `lw r8, 4(r9)`
- opcode = `100011` → Load Word  
- rs = 9 (base), rt = 8 (destination)  
- immediate = 4  

---

### `ad280008` → `sw r8, 8(r9)`
- opcode = `101011` → Store Word  
- rs = 9, rt = 8  
- immediate = 8  

---

### `11090004` → `beq r8, r9, 4`
- opcode = `000100` → Branch Equal  
- Branch taken if `r8 == r9`  

---

### `21280005` → `addi r8, r9, 5`
- opcode = `001000` → Add Immediate  
- r8 = r9 + 5  

---

### `11080004` → `beq r8, r8, 4`
- Always true → branch is taken  

---

## ⚠️ Invalid / Dummy Instruction

### `0000000B`
- opcode = `000000` (R-type)  
- funct field is undefined  
- Executes default ALU behavior  

---

## 🎯 Summary

- **Instruction Memory** → stores program instructions  
- **Register File** → stores operands and results  
- **Data Memory** → stores runtime data  

- Instruction decoding is performed using **opcode** and **funct fields**, which determine the operation executed by the processor.
